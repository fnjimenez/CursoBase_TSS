<h1 id="teoría-de-sistemas-y-simulación---semana-6">TEORÍA DE SISTEMAS Y SIMULACIÓN - SEMANA 6</h1>
<h2 id="unidad-3-simulación-y-generación-de-variables-aleatorias-continuación">UNIDAD 3: SIMULACIÓN Y GENERACIÓN DE VARIABLES ALEATORIAS (Continuación)</h2>
<p>En la semana anterior se establecieron las bases de la simulación como metodología y se introdujo el concepto de números pseudoaleatorios, así como las pruebas para verificar su uniformidad e independencia. Esta semana se avanza hacia la modelización de las entradas de un sistema real. A partir de datos observados, se deben identificar las distribuciones de probabilidad que mejor los representan, para lo cual se utilizan las pruebas de bondad de ajuste. Una vez seleccionada una distribución, es necesario generar, dentro del simulador, variables aleatorias que sigan dicha distribución. Esta semana se presentan los métodos fundamentales para la generación de variables aleatorias, tanto continuas como discretas, a partir de números uniformes, completando así el ciclo que va desde los datos reales hasta su incorporación en un modelo de simulación.</p>
<p><img src="image.png?conceptos_clave" alt=""> <em><strong>Conceptos clave</strong></em></p>
<ul>
<li><strong>Prueba de bondad de ajuste:</strong> Prueba estadística utilizada para determinar si un conjunto de datos observados sigue una distribución de probabilidad teórica específica.</li>
<li><strong>Prueba de Kolmogorov-Smirnov (K-S):</strong> Prueba no paramétrica que compara la función de distribución acumulada empírica con la de una distribución teórica.</li>
<li><strong>Prueba de Anderson-Darling (A-D):</strong> Prueba no paramétrica, similar a K-S, pero que da más peso a las colas de la distribución.</li>
<li><strong>Prueba de Chi-Cuadrada (<span class="katex--inline"><span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><msup><mi>χ</mi><mn>2</mn></msup></mrow><annotation encoding="application/x-tex">\chi^2</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height: 1.00855em; vertical-align: -0.19444em;"></span><span class="mord"><span class="mord mathnormal">χ</span><span class="msupsub"><span class="vlist-t"><span class="vlist-r"><span class="vlist" style="height: 0.814108em;"><span class="" style="top: -3.063em; margin-right: 0.05em;"><span class="pstrut" style="height: 2.7em;"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight">2</span></span></span></span></span></span></span></span></span></span></span></span>):</strong> Prueba de bondad de ajuste que compara frecuencias observadas con frecuencias esperadas en categorías o intervalos.</li>
<li><strong>Prueba de Shapiro-Wilk:</strong> Prueba paramétrica diseñada específicamente para contrastar la hipótesis de normalidad de un conjunto de datos.</li>
<li><strong>Método de la transformada inversa:</strong> Método fundamental para generar variables aleatorias a partir de un número uniforme, basado en la función de distribución acumulada inversa.</li>
<li><strong>Método de aceptación-rechazo:</strong> Método para generar variables aleatorias cuando la función de distribución acumulada no es fácilmente invertible.</li>
<li><strong>Método de Box-Müller:</strong> Método específico y eficiente para generar pares de variables aleatorias normales estándar independientes.</li>
</ul>

<h3 id="pruebas-de-bondad-de-ajuste-paramétricas-y-no-paramétricas">3.4 Pruebas de Bondad de Ajuste: Paramétricas y No Paramétricas</h3>
<p>Una vez que se ha verificado la aleatoriedad de los números base uniformes, el siguiente paso en un estudio de simulación es modelar las variables de entrada del sistema real (ej. tiempo entre llegadas de camiones, tiempo de servicio de una máquina, demanda de un producto). Para ello, se recolectan datos del sistema real y se busca una distribución de probabilidad teórica que se ajuste adecuadamente a esos datos. Las pruebas de bondad de ajuste son las herramientas estadísticas que nos permiten evaluar objetivamente la calidad de este ajuste (Ripley, 2009, p. 85).</p>
<h4 id="prueba-de-kolmogorov-smirnov-k-s">Prueba de Kolmogorov-Smirnov (K-S)</h4>

<h4 id="prueba-de-anderson-darling-a-d">Prueba de Anderson-Darling (A-D)</h4>
<p>La prueba de Anderson-Darling es una modificación de la prueba K-S que da más peso a las colas de la distribución. Es particularmente útil cuando es importante detectar desviaciones en los extremos de la distribución (ej. para modelar eventos extremos o valores atípicos, como pedidos de gran tamaño o averías de maquinaria). También es una prueba <strong>no paramétrica</strong> (Lewis &amp; McKenzie, 2017, p. 188).</p>
<p><strong>Ejemplo en Cadena de Suministro (Demanda de Productos):</strong><br>
Un planificador de demanda analiza 200 observaciones de demanda semanal de un producto. Le preocupa especialmente la precisión en los picos de demanda (colas de la distribución), ya que estos generan faltantes. Utiliza la prueba A-D para evaluar si una distribución normal es adecuada.</p>
<ul>
<li>La prueba A-D, al dar más peso a las colas, será más sensible para detectar si la demanda real tiene colas más pesadas (mayor probabilidad de valores extremos) que una distribución normal. Si la prueba rechaza la normalidad, el planificador optará por una distribución como la lognormal o la gamma, que modelan mejor los valores extremos.</li>
</ul>

<h3 id="generación-de-variables-aleatorias-continuas-y-discretas">3.5 Generación de Variables Aleatorias Continuas y Discretas</h3>
<p>Una vez que se ha seleccionado y validado una distribución de probabilidad para modelar una entrada, el siguiente paso es generar números aleatorios en el simulador que sigan esa distribución específica. Para ello, se parte de los números pseudoaleatorios uniformes <span class="katex--inline"><span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>U</mi><mo>∼</mo><mo stretchy="false">[</mo><mn>0</mn><mo separator="true">,</mo><mn>1</mn><mo stretchy="false">]</mo></mrow><annotation encoding="application/x-tex">U \sim [0,1]</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height: 0.68333em; vertical-align: 0em;"></span><span class="mord mathnormal" style="margin-right: 0.10903em;">U</span><span class="mspace" style="margin-right: 0.277778em;"></span><span class="mrel">∼</span><span class="mspace" style="margin-right: 0.277778em;"></span></span><span class="base"><span class="strut" style="height: 1em; vertical-align: -0.25em;"></span><span class="mopen">[</span><span class="mord">0</span><span class="mpunct">,</span><span class="mspace" style="margin-right: 0.166667em;"></span><span class="mord">1</span><span class="mclose">]</span></span></span></span></span> y se transforman en la distribución deseada (Ripley, 2009, p. 98).</p>

<p>Esta sexta semana ha completado el ciclo de preparación para la simulación. Se han presentado las herramientas estadísticas (pruebas de bondad de ajuste) que permiten validar la selección de distribuciones de probabilidad para modelar las entradas de un sistema a partir de datos reales. Una vez validadas, se han expuesto los métodos fundamentales (transformada inversa, aceptación-rechazo, Box-Müller, búsqueda en tabla) para generar, dentro del simulador, variables aleatorias que sigan fielmente esas distribuciones. Con estos elementos, se cuenta con la base necesaria para abordar la siguiente unidad, donde se aplicará el método Montecarlo, una técnica de simulación de amplio espectro para cuantificar el riesgo y la incertidumbre en la evaluación de escenarios y la toma de decisiones.</p>
